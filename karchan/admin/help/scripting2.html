Help on Commands

A command takes place as soon as a player enters that specific command. A command entered may consists of a regular expression. See for the regular expression syntax here.

Some examples:

quit

Will only match the exact word (and exact lowercasing) quit. Nothing else will do.

pounce (\w)+

Will only match a two word sentence starting with the word pounce. (For example : pounce karn, pounce bunnies, pounce everyone)

ask (to (\w)+ )?.+

A difficult one. The sentence must begin with ask, than an to [word] is possible but not mandatory and then an arbitrary number of keys.

It is also possible to make the command dedicated to a certain room. This means that the command will not be available outside of a certain room. The command in question must always point to a method that contains the scripting part responsible for executing the command. The scripting part comes very specific. Pay special attention to the method name, as that is the method the mud will search for in the database.

Scripting always starts as the following: 

<object>
<function name="command" params="person,command">
The parameters in the scripting part above need some explaining:
person
the person executing the command
command
an array containing the command. Every word is a specific element of the array. For example:

say to Karn Hello

Would provide the following:

  • command[0]="say"
  • command[1]="to"
  • command[2]="Karn"
  • command[3]="Hello"
  • command.length=4

Adding a command

Adding a command is quite simple. Just fill out the name of the command and the name of the method that you wish to call when that command is entered and press Add Command. It is possible to have the same command multiple times, for instance in different rooms. A command is created using the defaults. The defaults are:
  • a new id for the command
  • no room
  • default not callable (i.e. is not active)
Make sure that the method that this command is to call exists, or an error message will occur.

Note: When entering a regular expression to match what any one person will enter as a command, make certain that the regular expression is appropriate. This includes but is not limited to : the same number of closing parenthesis as opening parenthesis. Otherwise an exception will be generated in the log, and perhaps people will not be able to play the game any more.
