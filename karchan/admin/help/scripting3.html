Help on Methods

Methods in the database are quite simple and exist of a method name and the method source. The method name is used to refer to by special commands and events. The source is the tricky part.

Adding a method

Adding a method is quite simple. Just fill out the name of the method that you wish to create in the form and press Add Method. An empty method is created. Bear in mind that you will receive an error message if the method name is already taken.

Links

The source is encapsulated within XML tags. XML tags are kind of like HTML tags only more generic. Check here for an introduction. There is very little xml used in this mud, though. If you have problems writing xml code, consider using an xml editor. I have not looked around thoroughly, but there is a nice small java xml editor at http://jsxe.sourceforge.net.

One of the problems with XML is that HTML and XML do not mix well. Therefore, if you want to display HTML tags in your script, please use &gt; for > and &lt; for <. (Unless you are using an xml editor.)

If you wish to know more about the scripting language used, I suggest you check out the Simkin homepage. This is essential if you wish to implement scripts.

If you wish to know more about the % (percentage) keywords like %SNAME, please check here. Look for the three writeMessage commands.

Errors

If there is an error in one of the scripts you wrote, the proper way to find the error is to check out the Logs of the mmud. Every time a scripts is executed but fails, the error is written in the log.
